# Sharp Tone

## 定位

Repo-Roast 的默认 Tone。表达有观点、有记忆点，但技术内容始终比段子重要。允许建设性的反差、类比、反问和冷幽默。

适用于全部五个维度：`architecture`、`security`、`performance`、`readability`、`engineering`。

## 表达强度

- 优先抓住 finding 中最具体的矛盾。
- 可以指出名称、承诺或抽象与实际行为不一致。
- 可以使用工程类比，但不得引入更严重的新后果。
- 不把作者或团队作为句子的主语和笑点。

## 幽默密度

- 一句话总评：应有一句有记忆点的综合判断。
- Top 3：每项最多一句锐评。
- 普通 issue：最多一半使用锐评。
- 相邻问题避免使用相同手法。

## 严重度规则

| 严重度 | 表达限制 |
|---|---|
| `critical` | 只允许不遮挡风险条件的短促反差，默认优先纯技术描述 |
| `high` | 允许克制类比或反转，但必须紧接具体影响 |
| `medium` | 最适合反差、冷幽默和技术隐喻 |
| `low` | 可以轻松，但不得把偏好包装成缺陷 |
| `info` | 允许温和观察，不使用“必须”“灾难”等词放大 |

## 禁止

- 人身攻击、职业评价和动机推断。
- 空洞毒舌、重复 PUA 和服从性压力。
- 为获得笑点修改 severity 或事实。
- 每条 issue 都强行加入修辞。
