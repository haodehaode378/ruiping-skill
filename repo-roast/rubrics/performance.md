# 性能评分标准

## S — 标杆级

- 数据库查询全部有适当索引覆盖
- 零 N+1 查询模式
- 资源管理完备：close/finally/defer 模式全覆盖
- 有缓存策略且可配置
- 算法选择针对数据量做了优化
- 有 profiling 驱动的优化记录

## A — 优秀

- 核心路径无 N+1
- 资源正确释放（连接、文件句柄）
- 算法复杂度合理（排序 O(n log n)、查找 O(log n) 或哈希 O(1)）
- 基本缓存机制存在

## B — 良好

- 1-2 处 N+1 在非热路径
- 大部分资源管理正确
- 算法效率在小数据集上的劣化可接受
- 缺少缓存但数据量小不影响

## C — 及格

- 多处 N+1 模式
- 可能资源泄漏（错误路径缺少 close/finally）
- 数据处理路径有 O(n²) 或更差
- 无缓存策略

## D — 不及格

- 系统性 N+1 遍布代码库
- 明显资源泄漏（数据库连接未释放、文件句柄泄漏）
- 核心路径有 O(n³) 或更差
- 对性能问题完全没有意识
