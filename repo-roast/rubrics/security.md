# 安全评分标准

## S — 标杆级

- 零硬编码密钥/密码/token
- 所有外部输入使用白名单校验
- 全部数据库查询参数化
- CI 中有依赖漏洞扫描
- 认证/授权逻辑集中管理并经过审查
- 安全相关的 HTTP headers 配置正确

## A — 优秀

- 无硬编码密钥在源码中
- 外部输入有校验
- 数据库查询使用预处理语句
- 无 eval() 对用户输入
- 依赖版本锁定

## B — 良好

- 无明显密钥泄露在代码中
- 基本输入校验存在
- 大部分查询参数化
- 无明显注入点
- 少量依赖未锁定版本

## C — 及格

- 存在已确认但暴露范围有限的敏感配置问题
- 部分输入校验缺失
- SQL 字符串拼接可见
- eval()/exec() 作用于受控输入但缺少更窄的替代实现
- 多处依赖未锁定

## D — 不及格

- 硬编码密钥直接可见
- SQL/命令注入点容易利用
- 无输入校验
- eval() 直接作用在用户输入
- 密钥已进入版本控制历史
